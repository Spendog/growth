## üåä The Momentum Philosophy (Flow State)

> **"Mining is volatile. Memory is baggage. Flow is everything."**

We discovered that rigid "memory" (Trash Bins, Honey Pots) is a liability in a system where conditions change every 10 minutes.

### 1. No Trash Bins
*   **Old Way**: "Sector 42 was bad yesterday, so ban it today."
*   **New Way**: **Forgive immediately.** A sector that was dead 10 minutes ago might be the next goldmine. We never ban; we just move if it's cold *right now*.

### 2. No Honey Pots
*   **Old Way**: "Sector 7 gave a block last week, let's keep checking it."
*   **New Way**: **Chase the Flow.** If we are hitting high-diff shares *now*, we stay. If the flow stops, we move. We don't chase ghosts of past glory.

### 3. The "Cold Streak" Trigger
*   Instead of complex timers, we use a simple biological heuristic:
*   **"If I haven't eaten in 3 minutes, I move."**
*   This creates a natural, fluid exploration pattern that mimics a predator in a shifting environment.

## üìì The Lab Notebook Philosophy (Data Integrity)

We treat data with the same rigor as a scientific experiment.

### 1. Transient Noise (The Whiteboard)
*   **What**: Debug logs, connection attempts, raw packet dumps (`smart_miner.log`, `bun_crash.log`).
*   **Lifecycle**: **Ephemeral**. Wiped clean on every startup.
*   **Why**: A cluttered workspace hides the signal. We need a fresh slate for every session.

### 2. Permanent Signal (The Notebook)
*   **What**: High-value discoveries, strategic shifts, AI decisions (`precision_events.log`, `pattern_history.log`).
*   **Lifecycle**: **Immortal**. Never deleted, only appended.
*   **Why**: This is the "growth" of the system. Even if the machine reboots 1000 times, the history of its discoveries remains intact.

### 3. The "Startup Promise"
Every time the system boots, it explicitly validates this contract:
```text
[2/4] Archiving old logs...
   -> Archived: smart_miner.log (Transient)
   -> PRESERVING: precision_events.log (Master Record)
```
This ensures that **no data is ever lost by accident**, and **no noise is ever kept by habit**.

**Goal**: Make the system work for any search problem, not just Bitcoin.

**Example**:

**Goal**: Coordinate multiple machines as one distributed swarm.

**Technology**:

**Outcome**: 10 machines explore 1000 sectors simultaneously.

**Goal**: Prove the system works for a real scientific problem.
